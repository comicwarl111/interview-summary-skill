# interview-summary-skill
# 功能简介

**中文用户访谈小结结构化 skill** 用于将中文用户访谈逐字稿、访谈提纲、研究员笔记、历史小结表或多位受访者材料，整理成用户研究报告可直接使用的结构化访谈小结。

该 skill 会严格基于用户提供的附件内容进行归纳，不联网、不补充外部信息、不编造受访者未提及的内容。输出默认生成为 Excel `.xlsx` 文件，采用“研究维度 × 受访者”的横向对比矩阵，方便后续进行多用户比较、提炼洞察和撰写调研报告。

默认整理内容包括：用户背景、使用场景、行为流程、需求差异、痛点问题、工具/竞品评价、产品体验反馈、机会点、付费意愿及未使用原因等。

## 适用场景

- 需要把中文访谈逐字稿整理成结构化访谈小结。
- 需要将多个受访者材料合并到一个 Excel 小结表中，做横向对比。
- 需要按照访谈问题、调研目的或产品模块归纳用户反馈。
- 需要从长篇访谈中提炼用户场景、行为、需求、痛点和产品机会点。
- 需要生成用户研究报告前的中间分析材料。
- 需要参考已有小结 Excel 的格式，继续整理新访谈。
- 需要检查访谈内容是否被完整覆盖，避免遗漏重要信息。
- 适合用户研究、产品调研、可用性测试、需求访谈、竞品体验访谈等材料整理。

## 不适用场景

- 不适合没有任何访谈材料、只想凭空生成调研结论的任务。
- 不适合需要联网补充行业背景、市场数据或竞品资料的研究任务。
- 不适合要求写成完整咨询报告、商业分析报告或正式汇报稿的任务。
- 不适合逐字稿校对、转写纠错或音视频转写本身。
- 不适合需要保留每一句原话、不做提炼的逐字稿整理。
- 不适合医学、法律、金融等需要专业判断且超出访谈内容本身的结论生成。
- 不适合将研究员的引导语、假设或示例误当成受访者观点的整理任务。

## 输入材料

可处理的材料包括：

- 访谈逐字稿 / 转写稿 / 会议记录
- 访谈提纲 / 调研问题 / 研究目标
- 研究员笔记 / 观察记录
- 已有访谈小结 Excel / 历史小结样本
- 多位受访者的访谈材料
- 项目背景、产品介绍、功能说明等上下文材料

## 输出形式

默认输出为 Excel `.xlsx` 文件。

常见结构为：

| 维度 | 受访者 A | 受访者 B | 受访者 C |
|---|---|---|---|
| 基本信息 | ... | ... | ... |
| 使用场景 | ... | ... | ... |
| 使用流程 | ... | ... | ... |
| 痛点问题 | ... | ... | ... |
| 产品反馈 | ... | ... | ... |
| 付费意愿 | ... | ... | ... |

如果用户只提供单个受访者，也会使用同样的矩阵结构，方便后续继续追加其他受访者。

## 核心原则

- 只基于用户提供的材料整理。
- 不联网，不使用外部资料补充。
- 不编造、不推测受访者未表达的信息。
- 覆盖所有访谈中的有效信息。
- 对重复内容进行合并提炼。
- 区分受访者观点和研究员提问。
- 保持语言精简、具体、可比较。
- 优先输出 Excel 文件，而不是聊天中的长篇 Markdown。

## 使用方式

在Codex或其他AI工具中安装或引用该 skill 后，上传访谈逐字稿、访谈提纲、研究员笔记或历史小结表，并说明希望整理的范围。

示例提示词：

请使用 interview-summary skill，将这些中文访谈材料整理成可横向比较的用户研究访谈小结，并输出 Excel 文件。
